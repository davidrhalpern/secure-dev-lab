# Secure Dev Lab

## Purpose

This repository is a sandbox for learning GitHub workflows and secure software development lifecycle (SDLC) fundamentals.

It is intentionally simple and focused on:

- Practicing issue-driven development
- Using branches for controlled changes
- Opening and merging pull requests
- Tagging releases
- Building workflow confidence

## Learning Goals

The goal of this repository is not to build complex software.

The goal is to become comfortable with:

- Version control discipline
- Change traceability
- Transparent development practices
- Release hygiene
- Security-aware workflow habits

## Philosophy

Before building serious systems, we master the mechanics.

This repository exists to build those mechanics.
